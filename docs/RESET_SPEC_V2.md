# Compound Atlas V2 Reset Spec

**Date:** 2026-03-13  
**Author:** Planner (Swarm)  
**Context:** Current stack feels like AI slop. This is a ruthless product reset.

---

## Diagnosis: What Went Wrong

The current codebase suffers from **feature bloat without product clarity**. Symptoms:

1. **AI theater** — AI stack builder that generates recommendations from a database we don't have evidence for yet
2. **Community analytics vapor** — "prevalence scores" and "co-occurrence" metrics computed from... what community?
3. **Nootropic tunnel vision** — hardcoded nootropic templates, goal slices, category views everywhere
4. **Premature optimization** — stale compound detection, freshness SLAs, experience-level gating before we have users
5. **Competing stack creation flows** — manual builder, AI builder, templates, seed scripts
6. **IA debt masquerading as roadmap** — the backlog doc lists surface fixes, not product thesis

**Root cause:** Built features before defining the core user journey.

---

## Product Thesis (V2)

**CompoundAtlas is a Wikipedia for enhancement compounds, not a recommendation engine.**

### What It Is
- **Evidence database** — compounds with real studies, scored by research quality
- **Protocol notebook** — users build and track their own stacks/cycles
- **Research index** — PubMed/Semantic Scholar ingestion with evidence grading

### What It Is NOT
- Not a supplement store
- Not an AI advisor
- Not a social network
- Not a clinical decision support tool
- Not a nootropics-only site

### Core User Journey (V2)
1. **Discover** → Browse/search compounds by category, evidence score, legal status
2. **Learn** → Read compound detail pages with studies, mechanisms, dosing, interactions
3. **Plan** → Build a stack manually with interaction warnings
4. **Track** → Log doses and outcomes in a cycle
5. **Iterate** → Review cycle data, adjust protocol

That's it. No AI, no community features, no freshness theater.

---

## Architecture Principles (V2)

### 1. Data First, Features Second
- Invest in **compound data quality** (YAML files + ingestion pipeline) before UX polish
- Evidence scores must be **transparent and reproducible** — show the formula, not just a number
- Studies should link to **actual PDFs** when available (via PubMed Central, Sci-Hub mirrors, or preprints)

### 2. No ML Theater
- No AI stack generation until we have:
  - 100+ compounds with real evidence scores
  - 1000+ studies ingested
  - User feedback showing manual stack building is insufficient
- No "community analytics" until we have actual users sharing protocols

### 3. One Way to Do Things
- **One stack builder** (manual, form-based)
- **One cycle tracker** (simple daily log)
- **One research view** (studies table, filterable)
- Remove: AI builder, templates, seed-stacks script, community analytics

### 4. Legal and Safety Clarity
- Every compound page shows **legal status** (US-centric for now, expandable)
- **No dosing recommendations** — show study doses, not prescriptive guidance
- Interaction warnings are **informational**, not clinical advice
- Disclaimer on every page: "For research purposes only. Not medical advice."

### 5. Open Source Legitimacy
- **Compound YAML files** are the source of truth — easy to audit, easy to contribute
- **Evidence scoring algorithm** lives in code with tests, not hidden in LLM prompts
- **Research pipeline** is transparent (see `packages/research-ingestion/`)

## What to Kill (Immediate)

### Code to Delete
- `app/stacks/ai/` — AI stack builder
- `lib/stack-generator.ts` — programmatic stack generation
- `lib/community-analytics.ts` — prevalence/co-occurrence theater
- `lib/nootropic-templates.ts` — hardcoded templates
- `lib/stack-metadata.ts` — template-driven metadata
- `lib/compound-freshness.ts` — premature optimization
- `prisma/seed-stacks.ts` — autogenerated stacks
- `app/api/ai/generate-stack/` — AI generation endpoint
- `app/api/community/analytics/` — analytics endpoint
- `app/api/cron/generate-stacks/` — autogenerated stack seeding
- `docs/IA_BACKLOG_2026Q1.md` — surface-level feature audit

### Database Fields to Deprecate (leave in schema, stop using)
- User biometrics (`sex`, `weightLbs`, `heightFt`, `heightIn`) — unnecessary
- Stack `variant` field — over-engineered
- Any "community" or "trending" computed fields

### Dependencies to Remove
- `@anthropic-ai/sdk` — no AI features yet

## What to Keep

### Core Data Model
- `Compound` — name, category, legal status, evidence score, dosing
- `Study` — PMID, DOI, title, abstract, study type, evidence level
- `CompoundStudy` — join table linking compounds to studies
- `Stack` — user-created protocols
- `StackCompound` — compounds in a stack with dose/frequency
- `Cycle` — tracking a protocol over time
- `CycleEntry` — daily logs
- `BloodworkPanel` — optional lab tracking
- `Mechanism`, `SideEffect`, `Interaction` — relational compound data

### UI Components Worth Keeping
- `CompoundCard` — summary card for browse
- `CompoundDetail` — evidence breakdown, studies table, dosing info
- `StackBuilder` — manual form for creating stacks
- `CycleTracker` — daily logging interface
- All shadcn UI primitives (button, card, table, etc.)

### Backend Worth Keeping
- Prisma ORM + schema
- NextAuth for simple email/GitHub login
- tRPC for type-safe API (though we may simplify routes)
- Research ingestion pipeline (`packages/research-ingestion/`)

---

## What V2 Should NOT Include

Until we have 50+ active users and real usage data:

- ❌ AI-powered anything
- ❌ Social features (upvotes, comments, profiles)
- ❌ Community analytics or "trending" compounds
- ❌ Onboarding wizards or personalized recommendations
- ❌ Mobile apps
- ❌ API for third-party developers
- ❌ Freshness SLAs or stale compound dashboards
- ❌ Multi-language support
- ❌ Dark patterns (growth hacks, engagement metrics)

## V2 Core Pages (Simplified)

### 1. Landing Page (`/`)
**Purpose:** Explain what CompoundAtlas is in 10 seconds  
**Content:**
- Hero: "Evidence-Based Compound Research Database"
- 3-column value prop: Research-backed, Open source, No paywalls
- Top 6 compounds by evidence score
- CTA: "Browse Compounds" → `/compounds`

**Remove:**
- All marketing fluff
- "Scoring factors" explainer (move to About page)
- "Popular stacks" section (no autogenerated stacks)

### 2. Compound Explorer (`/compounds`)
**Purpose:** Browse and filter compounds  
**Filters:**
- Category (enum dropdown)
- Legal status (checkboxes)
- Evidence level (A/B/C/D radio)
- Search (simple text match on name/aliases)

**Display:**
- Grid of `CompoundCard` (name, category, evidence score, study count)
- Sort: evidence score desc (default), name asc

**Remove:**
- "Stale" badges (premature)
- "Related stacks" count (no autogenerated stacks)

### 3. Compound Detail (`/compounds/[slug]`)
**Purpose:** Deep dive on one compound  
**Sections (tabs):**
- **Overview** — description, dosing (from studies, not prescriptive), half-life, legal status
- **Evidence** — studies table (title, type, year, sample size, outcome), evidence score breakdown
- **Mechanisms** — pathways and descriptions
- **Safety** — side effects table, interaction warnings
- **Stacks** — public stacks that include this compound (if any)

**Remove:**
- AI-generated summaries
- "Community prevalence" scores
- "Trending" indicators

### 4. Stack Builder (`/stacks/new`)
**Purpose:** Create a custom protocol  
**Flow:**
1. Search and add compounds (autocomplete)
2. Set dose, frequency, duration for each
3. System shows interaction warnings (from `CompoundInteraction` table)
4. Name the stack, set goal (dropdown), add description
5. Save (requires auth)

**Remove:**
- AI builder tab
- Template suggestions
- "Experience level" gating
- "Composite evidence score" (nice-to-have, not MVP)

### 5. Stack Gallery (`/stacks`)
**Purpose:** Browse user-created stacks  
**Filters:**
- Goal (enum dropdown)
- Public only (no private stacks visible to others)

**Display:**
- Grid of stack cards (name, goal, compound count, creator)
- Click → `/stacks/[slug]` detail

**Remove:**
- Autogenerated stacks
- Upvote/fork counts (no social features yet)
- "Trending" or "popular" sorts

### 6. Cycle Tracker (`/cycles`)
**Purpose:** Log and visualize a protocol over time  
**Features:**
- Create cycle from a stack or freeform
- Daily log: compounds taken (checkboxes), weight, subjective metrics (sliders), notes
- Charts: weight over time, metrics over time
- Bloodwork upload (optional, simple form)

**Remove:**
- Symptom heatmaps (over-engineered)
- PDF export (nice-to-have, not MVP)
- Public sharing of cycles (keep private)

### 7. Research Feed (`/research`)
**Purpose:** Browse ingested studies  
**Filters:**
- Study type (RCT, meta-analysis, etc.)
- Evidence level (A/B/C/D)
- Compound (dropdown)
- Search (title/abstract keyword)

**Display:**
- Table of studies (title, authors, journal, year, PubMed link)
- Click title → show abstract in modal

**Remove:**
- "Latest studies for compounds you follow" (no follows yet)
- Email digests

## Tech Stack Simplification

### Keep
- **Next.js 14** (App Router) — proven, good SEO
- **Prisma ORM** — type-safe, good DX
- **PostgreSQL** — relational data fits perfectly
- **shadcn/ui** — accessible, themeable components
- **NextAuth** — simple auth (email magic link + GitHub OAuth)
- **Tailwind** — fast styling
- **tRPC** — type-safe API (though we may reduce surface area)

### Remove
- **Redis/Upstash** — no caching layer needed yet (Vercel edge caching is fine)
- **Typesense/Meilisearch** — Postgres full-text search is sufficient for <1000 compounds
- **Anthropic SDK** — no AI features

### Simplify
- **tRPC routers** — collapse to 4 routers: `compound`, `stack`, `cycle`, `research`
- **API routes** — remove AI and cron endpoints, keep only auth and tRPC

## Anti-Patterns to Avoid in V2

1. **"But users might want..."** — No. Ship the simplest thing that works. Add features when users ask.
2. **"Let's keep it as a hidden feature"** — No. Delete it. Git history is your archive.
3. **"This could be useful later"** — No. YAGNI. Build it when you need it.
4. **"The AI feature is almost done"** — Sunk cost fallacy. Delete it.
5. **"We should add X before launch"** — No. Launch with 100 compounds and core CRUD. Polish later.
